*เวอร์ชัน:* 1.0.0  
*วันที่ปรับปรุง:* 16 มิถุนายน 2026  
*สถานะ:* ร่างเบื้องต้น (Draft)

## 📖 ภาพรวม (Overview)
*wichit2s* เป็นแพลตฟอร์มจัดการธุรกิจอีคอมเมิร์ซแบบครบวงจร (All-in-One E-commerce Management Platform) ที่ออกแบบมาเพื่อเชื่อมต่อและบริหารจัดการกระบวนการทางธุรกิจหลัก 4 ส่วน ได้แก่ ระบบขายหน้าเว็บ, ระบบคลังสินค้า, ระบบการตลาด และระบบขนส่ง ให้ทำงานร่วมกันอย่างราบรื่น ลดความซ้ำซ้อนของข้อมูล และเพิ่มประสิทธิภาพการดำเนินงานแบบ Real-time

---

## 🏗️ โครงสร้างระบบย่อย (Sub-systems Architecture)

### 1. ระบบขายหน้าเว็บ (Web Sales System)
ระบบส่วนหน้าสำหรับลูกค้าและร้านค้า เพื่อจัดการกระบวนการซื้อขายตั้งแต่การเลือกสินค้าจนถึงการชำระเงิน

*   *FR-WS-01: แคตตาล็อกสินค้า (Product Catalog)*  
    แสดงรายการสินค้าพร้อมรูปภาพ, รายละเอียด, ราคา, และสถานะสต็อกแบบ Real-time รองรับการค้นหาและกรองตามหมวดหมู่/แท็ก
*   *FR-WS-02: ตะกร้าสินค้าและระบบชำระเงิน (Cart & Checkout)*  
    รองรับการคำนวณราคาสุทธิ (รวมภาษี/ค่าส่ง) และเชื่อมต่อเกตเวย์การชำระเงินหลากหลายช่องทาง (QR Code, บัตรเครดิต, E-Wallet, โอนเงิน)
*   *FR-WS-03: ระบบจัดการสมาชิก (Customer Portal)*  
    ลูกค้าสามารถลงทะเบียน, ดูประวัติการสั่งซื้อ, จัดการที่อยู่จัดส่ง, และตรวจสอบคะแนนสะสมได้
*   *FR-WS-04: แดชบอร์ดผู้ดูแลร้านค้า (Admin Sales Dashboard)*  
    แสดงภาพรวมยอดขายรายวัน/รายเดือน, กราฟแนวโน้ม, และตารางจัดการสถานะคำสั่งซื้อ (Pending, Paid, Shipped, Completed, Cancelled)

### 2. ระบบคลังสินค้า (Inventory Management System)
ระบบหลังบ้านสำหรับควบคุมและตรวจสอบความถูกต้องของสินค้าคงคลัง

*   *FR-IM-01: การจัดการข้อมูลสินค้า (Product & SKU Management)*  
    รองรับการเพิ่ม/แก้ไข/ลบสินค้า, กำหนดรหัส SKU, Barcode, และจัดกลุ่มหมวดหมู่สินค้า
*   *FR-IM-02: การตัดสต็อกอัตโนมัติ (Real-time Stock Deduction)*  
    ตัดจำนวนสินค้าในคลังทันทีเมื่อสถานะการชำระเงินเป็น "สำเร็จ" เพื่อป้องกันการขายสินค้าเกินสต็อก (Overselling)
*   *FR-IM-03: การแจ้งเตือนสินค้าใกล้หมด (Low Stock Alert)*  
    ระบบแจ้งเตือนผ่าน Email หรือ In-app Notification เมื่อจำนวนสินค้าต่ำกว่าจุดสั่งซื้อใหม่ (Reorder Point) ที่กำหนดไว้
*   *FR-IM-04: การตรวจนับและปรับยอดสต็อก (Stock Audit & Adjustment)*  
    รองรับการทำ Stock Count, บันทึกเหตุผลการปรับยอด (เช่น สินค้าชำรุด, หาย, หรือรับเพิ่ม) พร้อมประวัติการเปลี่ยนแปลง (Audit Log)

### 3. ระบบการตลาด (Marketing System)
เครื่องมือสำหรับกระตุ้นยอดขายและรักษาลูกค้าเก่า

*   *FR-MK-01: ระบบคูปองและส่วนลด (Coupon & Discount Engine)*  
    สร้างโค้ดส่วนลดแบบกำหนดเองได้ (เช่น ลดเป็นจำนวนเงิน, ลดเป็นเปอร์เซ็นต์, ส่งฟรี) และตั้งเงื่อนไขการใช้งาน (เช่น ยอดซื้อขั้นต่ำ, ใช้ได้เฉพาะสินค้าบางหมวด)
*   *FR-MK-02: ระบบสมาชิกและคะแนนสะสม (Loyalty & Point System)*  
    กำหนดกฎการสะสมคะแนน (เช่น ซื้อ 100 บาท = 1 คะแนน) และอนุญาตให้ลูกค้าแลกคะแนนเป็นส่วนลดในการสั่งซื้อครั้งถัดไป
*   *FR-MK-03: แคมเปญ Flash Sale*  
    รองรับการตั้งเวลาเริ่ม-สิ้นสุดโปรโมชั่น, นับถอยหลัง (Countdown Timer) และจำกัดจำนวนสินค้าต่อแคมเปญ
*   *FR-MK-04: การสื่อสารกับลูกค้า (CRM Communication)*  
    ส่งอีเมลหรือ SMS อัตโนมัติสำหรับกิจกรรมต่างๆ เช่น ยินดีต้อนรับสมาชิกใหม่, แจ้งสถานะออเดอร์, หรือแจ้งเตือนโปรโมชั่นพิเศษ

### 4. ระบบขนส่ง (Logistics & Shipping System)
ระบบจัดการกระบวนการจัดส่งสินค้าจากคลังถึงมือลูกค้า

*   *FR-LS-01: การเชื่อมต่อผู้ให้บริการขนส่ง (Courier API Integration)*  
    เชื่อมต่อกับ API ของผู้ให้บริการขนส่งชั้นนำ (เช่น Kerry, Flash Express, Thailand Post, J&T) เพื่อดึงอัตราค่าส่งและสร้างออเดอร์ขนส่ง
*   *FR-LS-02: การสร้างและพิมพ์ใบปะหน้าพัสดุ (Shipping Label Generation)*  
    สร้างไฟล์ PDF สำหรับพิมพ์ใบปะหน้า (Support เครื่องพิมพ์ความร้อน 4x6 นิ้ว) พร้อม Barcode/Tracking Number อัตโนมัติ
*   *FR-LS-03: การติดตามสถานะพัสดุ (Tracking Status Sync)*  
    ดึงสถานะการจัดส่งจากผู้ให้บริการขนส่งมาอัปเดตในระบบของ wichit2s แบบอัตโนมัติ (เช่น รอรับสินค้า, กำลังจัดส่ง, ส่งสำเร็จ, ส่งไม่สำเร็จ)
*   *FR-LS-04: การคำนวณค่าขนส่งอัจฉริยะ (Smart Shipping Calculation)*  
    คำนวณค่าส่งตามน้ำหนักปริมาตร (Volumetric Weight), ระยะทาง (Zones), หรือเงื่อนไขร้าน (เช่น ส่งฟรีเมื่อซื้อครบ 500 บาท)

## ⚙️ ข้อกำหนดที่ไม่ใช่ฟังก์ชันการทำงาน (Non-Functional Requirements)

1. *ประสิทธิภาพ (Performance):* ระบบต้องรองรับผู้ใช้งานพร้อมกัน (Concurrent Users) ได้อย่างน้อย 5,000 ราย โดยเวลาตอบสนอง (Response Time) ไม่เกิน 2 วินาที
2. *ความปลอดภัย (Security):* 
   - ข้อมูลการสื่อสารทั้งหมดต้องเข้ารหัสผ่าน HTTPS/TLS 1.3
   - รหัสผ่านของผู้ใช้ต้องถูก Hash ด้วยอัลกอริทึมที่ปลอดภัย (เช่น bcrypt)
   - เป็นไปตามมาตรฐาน PDPA (พระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล)
3. *ความพร้อมใช้งาน (Availability):* ระบบต้องมีความพร้อมใช้งาน (Uptime) ไม่ต่ำกว่า 99.9% ต่อเดือน
4. *ความสามารถในการขยายตัว (Scalability):* สถาปัตยกรรมต้องรองรับการเพิ่มจำนวนสินค้าและปริมาณการสั่งซื้อได้โดยไม่จำเป็นต้องปรับโครงสร้างระบบหลัก

## 🔧 การตั้งค่าและรันระบบด้วย Django + Uvicorn
1. ติดตั้ง dependencies ใน virtual environment:
   
   .\.venv\Scripts\Activate.ps1
   python -m pip install -e .
   
2. สร้างตารางฐานข้อมูลเริ่มต้น:
   
   python manage.py migrate
   
3. รันเซิร์ฟเวอร์ด้วย Uvicorn:
   
   uvicorn dw2069.asgi:application --reload --host 0.0.0.0 --port 8000
   
4. เปิดเว็บเบราเซอร์ไปที่ http://127.0.0.1:8000/

โครงสร้างโปรเจกต์นี้ใช้ Django เป็น web framework และ Uvicorn เป็น ASGI server เพื่อรองรับระบบแพลตฟอร์มที่ต้องการความเร็วและ concurrent handling
